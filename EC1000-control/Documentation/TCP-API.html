Returns: "0"

EC1000-control.lvlib:SM1000-control.lvclass:ExecuteJobContinuous.vi

Starts the execution of a job and repeats it forever.

The host must have taken control of the EC1000 using the 'TakeHostControl' command, and must have previously loaded a job from local ('LoadFlashJob') or USB based (LoadUSBJob) Flash storage prior to issuing this command. Job execution will begin immediately. If job execution is required to be synchronous with an external input such as STRTMRK, then it should have been constructed with a 'WaitForIO' instruction after the 'BeginJob' instruction. At the completion of the job, the job will loop until an 'Abort' command is received. The command returns as soon as the job is dispatched.

Command: <"ExecuteJobContinuous">,<job-name> or 208,<job-name>

Returns: "0"s

EC1000-control.lvlib:SM1000-control.lvclass:ExecuteJobOnce.vi

Starts the execution of a job one time.

The host must have taken control of the EC1000 using the 'TakeHostControl' command, and must have previously loaded a job from local ('LoadFlashJob') or USB based (LoadUSBJob) Flash storage prior to issuing this command. Job execution will begin immediately without waiting unless it was constructed with a 'WaitForIO' instruction. The job can be stopped at any time by issuing an 'Abort' command.

Command: <"ExecuteJobOnce">,<job-name> or 207,<job-name>

Returns: "0"s

EC1000-control.lvlib:SM1000-control.lvclass:OpenCOMPort.vi

Opens the specified serial I/O COM port on the EC1000.

This command is available only if the system is configured for accepting streaming job data over Ethernet.

The specified COM port is opened and is available for serial I/O.

This operation is intended to permit out-of-band communication to serial port based automation devices or laser systems.

A normal configuration might be specified as: "OpenCOMPort,2,38400,8,None,1,None.

Only COM port-ID 1 has hardware flow control support.

Command: <"OpenCOMPort">,<port-ID>,<baud-rate>,<data-bits>,<parity>,<stop-bits>,<flow-control> or 14,<port-ID>,<baud-rate>,<data-bits>,<parity>,<stop-bits>,<flow-control>

Returns: "0"

Releases exclusive control of the EC1000 to the LANStream host interface.

The host must have exclusive control of the EC1000 ('TakeHostControl') before issuing this command. Control is returned to the LANStream interface such that jobs may again be streamed to the EC1000 via that interface.

Command: "ReleaseHostControl" or 3

Returns: "0"
